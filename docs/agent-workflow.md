# EngX v2 — AI Agent Workflow Guide

This document defines how AI coding agents (GitHub Copilot, Claude, GPT-4, etc.) should operate within this repository.

## Before Making Any Change

1. **Read the relevant docs** — check `docs/architecture.md`, `docs/development-rules.md`, and `docs/product-scope.md` first.
2. **Understand the current state** — run `npm run build` to confirm the baseline is passing.
3. **Scope your change** — only modify what is necessary for the task.

## Making Changes

### Adding a New Feature
Follow `docs/feature-template.md` step by step.

### Modifying Existing Code
- Never rename or move files without updating all imports.
- Never change the Drizzle schema without generating a migration.
- Never add `"use client"` to a file that doesn't need it.

### Adding Dependencies
- Run `npm install <package>` — never manually edit `package.json` dependencies.
- Check for known vulnerabilities with the GitHub Advisory Database before adding.

## Verification

Before declaring a task complete, you MUST:

1. Run `npm run build` and fix ALL TypeScript and build errors.
2. Verify that no `any` types were introduced.
3. Confirm all new files follow naming conventions.
4. Confirm no secrets or credentials were committed.

## What NOT to Do

- ❌ Do NOT delete existing files unless explicitly instructed.
- ❌ Do NOT modify files in `src/components/ui/` (these are auto-generated by shadcn).
- ❌ Do NOT commit `.env.local` or any file with real credentials.
- ❌ Do NOT add `console.log` statements to production code.
- ❌ Do NOT use `@ts-ignore` or `any` without explicit justification in a comment.
- ❌ Do NOT skip the build verification step.

## Git Commits

Format: `type: short description`

Types: `feat`, `fix`, `chore`, `docs`, `refactor`, `test`

Example: `feat: add step submission with numeric tolerance checking`

Always include the Co-authored-by trailer:
```
Co-authored-by: Copilot <223556219+Copilot@users.noreply.github.com>
```
